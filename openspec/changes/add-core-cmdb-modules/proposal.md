# Change: 添加核心 CMDB 模块

## Why

本项目是一个全新的运维 CMDB 平台，需要建立核心功能模块以支持资产管理、关系管理、配置快照、审计和 WebSSH 等基础能力。这些模块是平台的基础，后续所有功能都将基于这些核心模块进行扩展。

## What Changes

- **新增 Assets 模块**：提供服务器、虚拟机、网络设备、应用等资产的统一登记和管理能力
- **新增 Relations 模块**：支持资产间依赖关系、拓扑关系的记录和查询
- **新增 Configs 模块**：实现资产配置快照的保存、版本管理和历史回溯
- **新增 Audit 模块**：记录所有操作的审计日志，支持操作追溯
- **新增 WebSSH 模块**：提供基于 Web 的 SSH 终端访问能力，支持多会话管理和操作审计
- **新增用户管理模块**：提供用户账户的创建、查询、更新、删除等基础管理能力
- **新增用户群组模块**：支持用户群组的创建和管理，便于批量权限分配
- **新增授权管理模块**：实现基于 RBAC 的权限控制，支持角色、权限和资源访问控制
- **建立项目基础架构**：采用 Clean Architecture 模式，建立后端分层结构（api/service/repository/model）
- **建立前端基础结构**：采用模块化组件结构，为各功能模块提供前端界面

## Impact

- **新增能力**：
  - `assets` - 资产管理能力
  - `relations` - 关系管理能力
  - `configs` - 配置快照能力
  - `audit` - 审计能力
  - `webssh` - WebSSH 能力
  - `user-management` - 用户管理能力
  - `user-groups` - 用户群组管理能力
  - `authorization` - 授权管理能力
- **受影响代码**：
  - 新建整个项目结构（后端 Go + 前端 React）
  - 数据库 schema 设计（PostgreSQL）
  - API 路由和接口定义
  - 前端页面和组件
